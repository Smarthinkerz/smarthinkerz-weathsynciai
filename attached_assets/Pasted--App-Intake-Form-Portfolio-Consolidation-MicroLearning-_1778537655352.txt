## 9. Payments

**How does it take money?**
- Per-user subscription model (monthly or annual billing)
- Payment processing via Smarthinkerz Tap Payments gateway
- Checkout flow: User selects plan → Quantity (number of seats) → Redirected to Smarthinkerz checkout page → Payment confirmation → Subscription activated

**Plans/SKUs sold (name + price + cycle):**
1. **Starter** - $3.95/user/month or $39.50/user/year
2. **Pro** - $8.95/user/month or $89.50/user/year
3. **Enterprise** - $12.00/user/month or $120.00/user/year

**Where does subscription state live?**
- `subscriptions` table (orgId, planSlug, billingCycle, status, renewalDate)
- `payments` table (orgId, amount, status, externalChargeId, paidAt)

**How is tier upgraded after payment?**
1. User completes payment on Smarthinkerz checkout page
2. Smarthinkerz sends webhook to `/api/webhooks/tap` with order ID
3. App calls `subscription.verifyPayment` with order ID
4. Payment status updated to "succeeded" in database
5. Subscription status updated to "active"
6. User gains access to plan features immediately
7. Email confirmation sent via Resend
8. Organization subscription updated with renewal date

## 10. Email / Notifications

**Provider:** Resend (email), Push API (push notifications), In-app toast (UI notifications)

**Email types sent:**
- Payment confirmation (after successful payment)
- System alerts (breach notifications, GDPR alerts, service status)
- Onboarding welcome email (after user creation)
- Lesson assignment notification (when lessons assigned)
- Owner notifications (form submissions, feedback, alerts)

**Who triggers them:**
- **Payment confirmation** - Triggered by `subscription.verifyPayment` after successful payment
- **System alerts** - Triggered by admin via `system.notifyOwner` mutation
- **Onboarding email** - Triggered by `onboarding.submitShiftSchedule` completion
- **Lesson assignment** - Triggered by `assignment.create` bulk endpoint
- **Owner notifications** - Triggered by user actions (new feedback, breach detection)

**Notification channels:**
- Email (Resend API)
- Push notifications (browser push, stored in `push_subscriptions` table)
- In-app toast (UI notifications via Sonner library)
- Audit logs (all actions logged for admin review)

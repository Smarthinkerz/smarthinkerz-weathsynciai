ROLE
Full-system audit of Nexus — the cross-app memory/context/orchestration layer for SmarThinkerz.
Nexus aggregates a user's most sensitive data across every app (HR/salary, financial, learning,
business, travel) in a User Intelligence Graph (vector DB + Postgres), serves it via getContext(),
and coordinates apps via an orchestration engine, multi-agent runtime, and event bus. A leak is
catastrophic and platform-wide, so this audit is weighted overwhelmingly toward isolation, auth,
and privacy.

EXECUTE-AND-PROVE (command → raw output → verdict). Where a claim has no test, WRITE and RUN one.
Deliver (A) executed evidence and (B) a final report with success/failure rate + recommendations.

PART A — EXECUTE & PROVE (one tier at a time, pausing for approval after each)

== TIER 1 — ISOLATION & AUTH ACROSS ALL FOUR LAYERS (Launch-Gating) ==
1. CROSS-USER ISOLATION (getContext API)
   As user A (or an app session acting for A), attempt to read AND mutate user B's memory/context
   via every path (IDs, query params, tokens, app identity). All attempts must be denied. Show
   requests + rejection responses.
2. VECTOR-DB PARTITIONING
   Seed users A and B with deliberately similar embeddings. Run user A's semantic query and prove
   zero of user B's vectors are returned. Confirm every vector query carries a per-user/tenant
   filter (Qdrant payload filter / Weaviate tenant) — not just app-level filtering.
3. EVENT-BUS SCOPING
   Prove an app only receives events for users/contexts it is authorized for. No user data should
   be broadcast to unsubscribed apps.
4. AGENT AUTHORIZATION
   Prove an agent registered by app X cannot act on a user who hasn't authorized X. Tokens must be
   scoped and non-replayable across users/apps.
5. INTER-APP AUTH
   Requests with no/invalid service+user credentials are rejected. An app cannot fetch context for
   a user who hasn't authorized it.
6. SECRET / DATA EXPOSURE
   Grep all responses and config files for secrets, internal IDs, or another user's data bleeding
   into payloads.

== TIER 2 — PRIVACY & GOVERNANCE ==
7. CONTEXT SCOPING (Least Privilege)
   Assert the exact field set returned by getContext() per app. The travel app must not receive
   HR/salary or financial fields. CoreHR data must not reach unrelated apps. (Field-level
   over-sharing, distinct from Tier 1 binary access denial.)
8. ENCRYPTION
   Sensitive data is encrypted at rest. TLS is enforced in transit. Show evidence.
9. CONSENT
   Cross-app data sharing requires and records explicit user consent. Nothing is shared before
   consent exists.
10. GDPR / DATA RIGHTS
    Full export (all of a user's memory across both stores) and hard deletion (real erasure across
    Postgres and the vector store, not a soft flag). Write and run tests for both.

== TIER 3 — INTEGRITY, ORCHESTRATION SAFETY, RESILIENCE ==
11. MEMORY INTEGRITY
    Writes are user-scoped and versioned. One app cannot corrupt or overwrite another app's
    namespace. Conflicting writes resolve deterministically.
12. ORCHESTRATION SAFETY + BLAST RADIUS
    Cross-app actions are authorized and bounded. The human-approval gate for privileged actions
    cannot be bypassed. A failed or malformed workflow (Temporal/BullMQ) cannot cascade unintended
    proactive actions across apps.
13. RESILIENCE / SPOF
    Simulate Nexus being unavailable. Apps must degrade gracefully and fail safe (no crash, no data
    exposure, no fail-open auth).
